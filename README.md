# 📊 インサイドセールス分析ダッシュボード

## 🌟 概要
架電データを可視化するインタラクティブなダッシュボードです。Zipファイルアップロード機能により、ユーザーが独自のJSONデータをアップロードして分析可能。認証機能付きで、月次トレンド、支店別・スタッフ別分析を提供します。

## 🔧 最近の修正 (2025年1月)

**変更内容**:

**メリット**:

- 外部サービスへの依存なし
- ユーザーがデータを完全にコントロール

## 🗂️ 新機能: Zipファイルアップロード

### データソースの変更
- **📁 Zipファイルアップロード**: ユーザーがJSONファイルをZip形式でアップロード
- **🔒 プライバシー保護**: データはサーバーに保存されず、セッション中のみ保持
- **📊 柔軟な分析**: 複数月のデータを一度にアップロード可能

### 対応ファイル形式
- `基本分析_YYYY-MM.json`
- `詳細分析_YYYY-MM.json`
- `月次サマリー_YYYY-MM.json`
- `定着率分析_YYYY-MM.json`

## 📈 機能

- **認証システム**: ユーザー名・パスワード認証

   - 支店別総架電数・成功率
   - ランキング表示

3. **👥 スタッフ別分析**
   - 架電数ランキング
   - 成約率ランキング
   - 個人別パフォーマンス

4. **📋 詳細データ**
   - フィルター機能（支店・スタッフ）
   - JSONデータの直接表示

## 🛠️ トラブルシューティング

### 📁 ファイルアップロード関連

#### Zipファイルがアップロードできない
**症状**: アップロードボタンを押しても反応がない

**原因**:
- ファイルサイズが制限を超えている
- ファイル形式が不正
- ブラウザのJavaScriptが無効

**解決策**:
- ファイルサイズを200MB以下に制限
- .zip形式のファイルを確認
- ブラウザのJavaScriptを有効化

#### JSONファイルが見つからない
**症状**: アップロード後「JSONファイルが見つかりませんでした」エラー

**原因**:
- Zipファイル内にJSONファイルが含まれていない
- ファイル名が対応形式と異なる

**解決策**:
- Zipファイル内にJSONファイルが含まれているか確認
- ファイル名が以下の形式になっているか確認：
  - `基本分析_YYYY-MM.json`
  - `詳細分析_YYYY-MM.json`
  - `月次サマリー_YYYY-MM.json`
  - `定着率分析_YYYY-MM.json`

### 🔐 認証関連

#### ログインできない
**症状**: ユーザー名・パスワードを入力してもログインできない

**解決策**:
- ユーザー名: `admin` / パスワード: `admin123`
- ユーザー名: `user` / パスワード: `user123`

## 📝 使用方法

### 1. データの準備
分析したいJSONファイルを以下の形式で準備：
- `基本分析_2024-09.json`
- `詳細分析_2024-09.json`
- `月次サマリー_2024-09.json`
- `定着率分析_2024-09.json`

### 2. Zipファイルの作成
準備したJSONファイルをZip形式で圧縮

### 3. アプリケーションの起動
```bash
streamlit run streamlit_app.py
```

### 4. データのアップロード
- 左サイドバーの「データアップロード」セクションでZipファイルをアップロード
- ドラッグ&ドロップまたはクリックでファイル選択

### 5. 分析の実行
- アップロード後、分析タイプを選択
- 月を選択して分析を開始
